# AI Ready Data

## 한 줄 정의
AI가 제대로 활용할 수 있도록 **품질·구조·맥락이 준비된 데이터**. 단순히 데이터를 쌓는 것이 아니라, AI가 이해하고 탐색할 수 있는 형태로 설계·관리된 상태.

---

## 왜 AI Ready Data가 중요한가

### 기업 AI 도입 실패의 패턴

> "RAG를 도입했는데 왜 안 되지? 우리 산내 데이터로 구성했는데, 윗분이 날씨를 물어봤다. 그건 안 된다." — 엔코아 현장 사례

기업들이 AI 도입 후 실망하는 구조:
```
[AI 기술 도입]
    ↓
데이터가 준비되지 않음 (품질 불량, 구조 불명확, 맥락 없음)
    ↓
AI가 제대로 작동하지 않음
    ↓
"AI가 별로네" 라는 오해
    ↓ (실제로는)
데이터 준비 문제
```

**87%의 기업**이 AI를 도입했거나 검토 중 (2026년 기준, 엔코아 조사).
하지만 대부분이 데이터 준비 없이 기술부터 얹는 실수를 반복.

## AI Ready Data의 구성 요소

### 1. 데이터 품질 (Data Quality)
- 정확성: 오류, 결측치, 중복 없음
- 완전성: 필요한 데이터가 다 있음
- 일관성: 같은 개념은 같은 방식으로 표현됨

### 2. 데이터 구조 (Data Structure)
- 온톨로지 기반 개념 정의
- 지식 그래프로 관계 구조화
- 그래프DB 또는 구조화된 저장소에 저장

### 3. 메타데이터 관리 (Metadata)
- 데이터가 무엇인지, 어디서 왔는지, 언제 만들어졌는지
- AI Agent가 데이터를 "찾고 이해"하기 위해 필수
- **Agentic Unified META**: AI Agent 운영에 최적화된 메타데이터 전략

### 4. 맥락 (Context)
- 데이터 간 관계와 업무 프로세스
- 암묵적 규칙·의사결정 흐름을 명시화
- → [[지식-그래프-디지털-맥락지도]]

## AI 기본법과 AI Ready Data

**한국 AI 기본법** (2026.01.22 시행, 실질 발효는 2027년):

| 의무 | 내용 | 데이터 관점 |
|------|------|-----------|
| 투명성 | AI 사용 여부 명시 의무 | 어떤 데이터로 AI를 작동시켰는지 추적 가능해야 함 |
| 데이터 품질 | 정확한 데이터 사용 책임 | AI Ready Data가 컴플라이언스 기반 |
| 보안 | 데이터 보호 | 민감 데이터 접근 통제 |
| 책임 | AI 결과에 대한 책임 | 결과 추적을 위한 처리 이력 필요 |

→ AI Ready Data는 좋은 AI 성능을 위한 조건이면서, **동시에 법적 의무 이행의 기반**이기도 하다.

## AX 프로젝트에서의 AI Ready Data 체크리스트

AX 전문가로서 고객사 AI 도입 전 확인해야 할 사항:

- [ ] **데이터 목록**: 어떤 데이터가 있는가?
- [ ] **데이터 품질**: 오류·결측·중복 수준은?
- [ ] **메타데이터**: 각 데이터의 의미와 출처가 정의되어 있는가?
- [ ] **관계 구조**: 데이터 간 관계가 명시되어 있는가?
- [ ] **온톨로지**: 업무 도메인의 개념 기준이 정의되어 있는가?
- [ ] **거버넌스**: 데이터 생성·수정·삭제 프로세스가 있는가?
- [ ] **컴플라이언스**: AI 기본법 대응 준비가 되어 있는가?

## 엔코아의 접근: 기본으로 돌아가기

> "예전에는 응용·분석 중심으로 눈에 보이는 것만 했다. 시행착오 후 기본적인 데이터 관리부터 제대로 해야 한다는 것을 이제 인식하고 있다." — 김선영

**AI 도입 순서:**
```
❌ 잘못된 순서: AI 기술 → 데이터 넣기 → 실망
✅ 올바른 순서: 데이터 관리 기본 → AI Ready Data 구축 → AI 기술 적용
```
