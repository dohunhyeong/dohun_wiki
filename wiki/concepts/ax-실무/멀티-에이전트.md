# 멀티 에이전트 (Multi-Agent)

## 한 줄 정의
여러 AI 에이전트가 각자 병렬로 작업을 수행하고, 사람이 전체 흐름을 감독하는 구조.

---

## 싱글 에이전트 vs 멀티 에이전트

| 구분 | 싱글 에이전트 | 멀티 에이전트 |
|------|-------------|-------------|
| 작업 방식 | 순차적, 단일 요청에 응답 | 여러 작업을 병렬로 동시 처리 |
| 사람의 역할 | 매 단계마다 직접 개입 | 전체 흐름 감독, 필요 시점에만 개입 |
| 복잡도 | 낮음 | 높음 (에이전트 간 조율 필요) |
| 적합 업무 | 단순 Q&A, 단일 태스크 | 다단계 복합 업무, 병렬 처리 가능 작업 |

---

## 멀티 에이전트 구조의 핵심 요소

1. **오케스트레이터(Orchestrator)**: 전체 작업 흐름을 조율하는 상위 에이전트
2. **서브에이전트(Sub-agent)**: 특정 작업을 전담하는 하위 에이전트
3. **Human-in-the-loop 지점**: 사람이 개입해야 하는 체크포인트
4. **상태 관리**: 어떤 에이전트가 무엇을 처리 중인지 추적하는 시스템

---

## 실제 사례: Anthropic Claude Code Agent View

Anthropic이 Claude Code에 추가한 **에이전트 뷰(Agent View)**는 멀티 에이전트 시대의 구체적 신호.

- 여러 Claude Code 세션을 하나의 인터페이스에서 관리
- 에이전트를 백그라운드로 보내고, 입력이 필요한 시점에만 사람이 개입
- AI 활용이 "질의응답"에서 "작업 위임 + 감독"으로 이동하고 있음을 보여줌

---

## AX에서 멀티 에이전트가 중요한 이유

기업 업무는 하나의 작업으로 끝나는 경우가 드물다.

> 예: 세금계산서 처리 → 공급자 정보 추출 → 내부 기준 검증 → 담당자 검토 → ERP 입력

이 흐름을 에이전트별로 분리하면:
- 각 단계를 전문화된 에이전트에게 위임 가능
- 병렬 처리로 처리 속도 향상
- 특정 단계에서만 사람이 확인 → 전체 효율 증가

**단, 자동화의 범위가 넓어질수록 통제와 검증의 설계가 더 중요해진다.**

---

## Dohun의 관점에서

AX 전문가로서 멀티 에이전트 설계는 "어떤 에이전트를 만드냐"보다 **"어떤 업무를 어떻게 분리하고, 어디에서 사람이 개입하게 할 것인가"** 를 설계하는 것.

도메인마다 답이 다를 수 있음:
- 선박: BL(Bill of Lading) 추출 에이전트 → OCR 기반
- 제조: 불량품 탐지 에이전트 → 객체 탐지 기반
- 금융: 계약서 검토 에이전트 → NLP + 규정 검증 기반

→ 핵심은 **도메인 이해 + 업무 프로세스 분해 + 적절한 AI 기술 선택**.
