# 지식 그래프 & 디지털 맥락지도

## 한 줄 정의
**지식 그래프(Knowledge Graph)**: 개념과 그 관계를 그래프 구조로 표현한 지식 체계.
**디지털 맥락지도**: AI Agent가 기업 데이터를 탐색할 때 사용하는, 지식 그래프 기반의 기업 데이터 관계·맥락 구조.

> "AI Agent가 제대로 일하려면, 기업 직원처럼 프로세스와 데이터 관계를 알아야 한다. 디지털 맥락지도가 그 지식을 제공한다." — 김선영, 엔코아

---

## 지식 그래프란

개념(노드)과 관계(엣지)를 그래프 형태로 연결한 구조.

```
[공급자] --[발행]--> [세금계산서] --[포함]--> [공급가액]
    |                                              |
[isA: 기업]                              [관련: 세액 계산]
```

- 온톨로지의 **구조적 표현** + 실제 데이터(**인스턴스**)가 결합된 형태
- 구글 지식 그래프, DBpedia 등이 대표적 사례
- 기업 내부에서는 업무 프로세스, 데이터 관계를 지식 그래프로 표현

---

## 디지털 맥락지도 (엔코아 개념)

엔코아가 제시하는 개념. AI Agent가 기업 데이터 환경을 탐색하기 위한 **지식 그래프 기반 맥락 구조**.

### AI Agent에게 맥락지도가 필요한 이유

```
[사람 직원]
기업에 입사 → 암묵적 지식 습득 (프로세스, 관계, 규칙) → 일을 제대로 수행

[AI Agent]
기업에 투입 → ??? → 일을 제대로 수행
```

AI Agent가 기업 데이터를 찾으려면:
1. 이 데이터가 **어디에** 있는지
2. 다른 데이터와 **어떤 관계**인지
3. 업무 **맥락**(암묵적 프로세스, 규칙)이 무엇인지

→ 이 모든 것을 담은 것이 **디지털 맥락지도**

### 구성 요소

| 구성 | 설명 |
|------|------|
| 데이터 범위 | AI가 접근할 수 있는 기업 데이터 목록 |
| 개념 구조 | 온톨로지 기반 개념 정의 |
| 관계 구조 | 데이터 간 관계 (지식 그래프) |
| 메타데이터 | 데이터의 의미, 출처, 갱신 주기 등 |
| 업무 맥락 | 암묵적 프로세스, 규칙, 의사결정 흐름 |

## AI Ready Data와의 관계

```
AI Ready Data (전략)
    ↓
온톨로지 설계 (의미 기준 정의)
    ↓
지식 그래프 / 디지털 맥락지도 구축 (관계 구조화)
    ↓
그래프DB 저장 (구현)
    ↓
AI Agent가 데이터를 탐색·이해 가능
```

## 그래프 모델링 접근법

엔코아가 제시하는 두 가지 방법:

### 1. 있는 그대로 모델링
기존 데이터를 그대로 그래프로 옮김. 빠르지만 복잡도가 높아질 수 있음.

### 2. 재구성(클러스터링) 후 모델링
비슷한 개념을 묶어(클러스터링) 단순화한 후 모델링.
- 큰 단위로 먼저 구성 → 내부에서 세부적으로 확장
- AI가 탐색하기 더 효율적인 구조

> **품질 좋은 맥락지도 = 단순하고 명확하게 재구성된 그래프**

## Dohun의 관점에서

VectorDB 기반 RAG를 만들 때 겪었던 문제(표 잘림, 청킹 전략)가 결국 **"데이터의 관계와 맥락을 구조화하지 않았기 때문"**이라는 것과 연결된다.

- VectorDB: 텍스트 의미 유사도 기반 → 관계 탐색에 약함
- 그래프DB + 지식 그래프: 관계 경로 탐색에 강함
- **GraphRAG**: 두 방식을 결합하여 더 정확한 컨텍스트 제공

AX 전문가로서 고객사의 데이터를 분석할 때, 먼저 **디지털 맥락지도**를 그리는 것이 AI 도입 성공의 핵심 선행 작업일 수 있다.
