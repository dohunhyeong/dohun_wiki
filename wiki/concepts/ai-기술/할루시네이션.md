# 할루시네이션 (Hallucination)

## 한 줄 정의
AI가 사실과 다른 내용을 자신감 있게 생성하는 현상. 모델의 성능 문제가 아닌 **의미 기준(해석 기준)의 부재**에서 비롯되는 경우가 많다.

---

## 두 가지 원인 구분

| 원인 유형 | 설명 | 해결 방향 |
|----------|------|----------|
| **모델 문제** | 학습 데이터 부족, 망각, 과신(overconfidence) | 모델 개선, 파인튜닝 |
| **설계 문제** | 의미 기준이 없어 확률적으로 해석 | 온톨로지 설계, RAG 구조 개선 |

> 한국딥러닝 관점: 기업 문서 AI에서 발생하는 할루시네이션의 핵심 원인은 **설계 문제** — 즉 의미 기준이 정의되지 않은 것.

---

## 기업 문서에서 발생하는 구체적 패턴

**같은 개념, 다른 표현 문제**
- "공급가액", "공급금액", "금액(부가세 제외)" → 온톨로지 없이는 AI가 확률로 해석
- 때마다 다른 필드를 참조 → 일관되지 않은 답변

**청킹 경계 문제**
- 표(Table)가 청킹 중에 잘리면 → 표의 의미 맥락이 단절
- 잘린 표를 참조한 AI → 불완전한 정보로 답변 → 할루시네이션

**RAG 검색 오류**
- 의미적으로 같은 내용인데 표현이 달라서 → 검색에서 누락
- 관련 문서를 못 찾고 다른 근거로 답변 생성

---

## 온톨로지로 해결하는 방식

```
[문제] "공급금액이 얼마야?" 
       ↓
[온톨로지 없을 때] "공급금액"으로만 검색 → 
                   "공급가액"으로 저장된 문서 누락 → 
                   틀린 답변 또는 "모른다"
       ↓
[온톨로지 있을 때] "공급금액" → 온톨로지에서 "공급가액" 개념과 동의어로 매핑 →
                   모든 관련 표현 포함하여 검색 →
                   정확한 답변
```

---

## AI 에이전트에서 더 치명적인 이유

단순 챗봇은 틀린 답변을 사람이 보정할 수 있다.  
AI 에이전트는 틀린 해석으로 **시스템에 직접 입력하거나 업무를 실행**한다.

> 의미 오류 → 업무 오류 → 실제 손실

이것이 [[관리-가능한-자동화]]에서 통제 가능성 설계가 중요한 이유이기도 하다.
