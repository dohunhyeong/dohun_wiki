# wiki/ 운영 규칙

## 역할
AI가 전적으로 작성하고 유지하는 지식 위키. Dohun은 읽고 방향을 잡는다.

## 하위 폴더
| 폴더 | 저장할 것 |
|------|-----------|
| `concepts/ai-기술/` | AI 기술 개념 (온톨로지, 할루시네이션, RAG, LLM, 임베딩 등) |
| `concepts/ax-실무/` | AX 적용 방법론 (멀티에이전트, 기업문서자동화, 관리가능한자동화 등) |
| `sources/` | 인제스트된 소스 요약 (왜 읽었는지, 무엇을 얻었는지) |
| `projects/` | AX 프로젝트별 위키 (문제, 접근법, 결과, 교훈) |
| `self/` | Dohun 자신에 대한 지식 (결정 패턴, 회고, 강점/약점) |

## 개념 분류 기준
- **ai-기술/**: "이 기술은 무엇이고 어떻게 작동하는가" → 이해 레이어
- **ax-실무/**: "어떻게 설계하고 조직에 적용하는가" → 방법론 레이어
- 경계가 애매하면 ai-기술/에 먼저 두고, AX 적용 관점이 쌓이면 ax-실무/로 이동

## 페이지 생성 기준
- 새 소스 인제스트 시 → `sources/`에 요약 페이지 생성
- 3회 이상 언급된 개념 → 적절한 `concepts/` 하위 폴더에 생성
- 가치 있는 쿼리 답변 → 새 페이지로 저장
- 모든 페이지는 `[[관련 페이지]]` 링크 포함

## 필수 파일
- `index.md` — 전체 위키 목록. 질문 시 가장 먼저 참조.
- `log.md` — append-only 작업 이력.
