# Ingest 워크플로우

오늘 날짜: !`date +%Y-%m-%d`

---

## Step 0. 대상 파일 결정

**`$ARGUMENTS`가 있으면** `raw/` 하위에서 해당 파일명을 검색해 읽는다.

**없으면** `raw/` 하위 모든 `.md` 파일 중 `ingest_status: "pending"` 인 것을 목록으로 보여주고, 어느 파일을 인제스트할지 사용자에게 확인한다.

> ⚠️ **raw/ 파일은 절대 수정하지 않는다. 읽기만 한다.**

---

## Step 1. 소스 요약 제시

파일을 읽고 다음 형식으로 정리해 사용자에게 보여준다.

```
## 📄 소스 요약

**출처**: 저자/기관 | 발행일
**URL**: 원문 링크

### 🔑 핵심 주장 (Key Claims)
1. ...
2. ...

### 🏷️ 언급된 엔티티
| 유형 | 항목 |
| 조직 / 도구 / 인물 / 기술 | ... |

### 💡 다루는 개념
- 개념1, 개념2, ...
```

---

## Step 2. 3가지 질문

요약 제시 후 반드시 다음 3가지를 사용자에게 질문한다.
**답변을 받기 전까지 wiki 파일 작업을 절대 시작하지 않는다.**

1. **이 글을 왜 캡처했어?**
2. **지금 하고 있는 일과 어떻게 연결돼?**
3. **이걸로 뭘 해보고 싶어?**

---

## Step 3. wiki/ 파일 생성

사용자 답변을 받은 후 다음 순서로 진행한다.

### 3-1. 소스 요약 페이지

- **경로**: `wiki/sources/<slug>.md`
- **slug 규칙**: 소스 핵심 주제 + 저자명, 한국어 + 영문 혼용 가능
  - 예: `AI는왜틀리는가-온톨로지-한국딥러닝`, `멀티에이전트-기업문서자동화-한국딥러닝`
- **포함 내용**: 출처 정보, 왜 저장했나, 핵심 주장, 엔티티, 나의 경험 연결, 관련 페이지

### 3-2. 개념 페이지

- **새 개념이면** → 아래 분류 기준에 따라 적절한 폴더에 생성
- **기존 페이지와 관련 있으면** → 해당 페이지에 섹션을 추가하거나 업데이트
- **소스 추가 시** 페이지 frontmatter의 `sources` 카운트를 +1 한다

### 3-3. 개념 분류 기준

현재 `wiki/concepts/` 구조:

```
wiki/concepts/
├── ai-기술/    ← "이 기술은 무엇이고 어떻게 작동하는가" (이해 레이어)
│               예: 온톨로지, 할루시네이션, RAG, LLM, 임베딩, 프롬프트엔지니어링...
└── ax-실무/    ← "어떻게 설계하고 조직에 적용하는가" (방법론 레이어)
                예: 멀티에이전트, 기업문서자동화, 관리가능한자동화, 도메인별AI...
```

**경계가 애매하면** → `ai-기술/`에 먼저 생성, AX 적용 관점이 쌓이면 `ax-실무/`로 이동.

**새 서브폴더 판단 기준:**
기존 두 분류에 맞지 않는 새 주제군(예: 통계, 보건, HR, 교육 등)에 해당하는 개념이 3개 이상 생길 것으로 보이면, 사용자에게 새 서브폴더 생성을 제안한다. 사용자 확인 후 생성한다.

## Step 4. 페이지 형식

모든 wiki 페이지는 다음 형식을 따른다:

```markdown
---
type: concept | source-summary | project | person | self
tags: []
created: YYYY-MM-DD
updated: YYYY-MM-DD
sources: 0
---

# 페이지 제목

본문...

## 관련 페이지
- [[링크1]]
- [[링크2]]
```

- 모든 페이지는 `[[페이지명]]` 형식으로 관련 페이지를 링크한다.
- 사용자(Dohun)의 경험·인사이트가 있으면 별도 섹션으로 강조해서 기록한다.
- 아직 모르는 것은 **빈 곳 (아직 모르는 것)** 섹션에 명시한다.

---

## Step 5. index.md 업데이트

`wiki/index.md`에서:
- 소스 요약 → "소스 요약 (sources/)" 테이블에 행 추가
- 개념 페이지 → 해당 분류 섹션 테이블에 행 추가
- 총 페이지 수, 마지막 업데이트 날짜(오늘) 갱신

---

## Step 6. log.md 업데이트

`wiki/log.md` 기존 첫 번째 `##` 항목 바로 위에 삽입 (append-only):

```
## [YYYY-MM-DD] ingest | 소스 제목
- 출처: 저자/기관 (발행일)
- raw: `raw/경로/파일명.md`
- 캡처 이유: (Q1 답변 요약)
- 신규 페이지: [[페이지1]], [[페이지2]]
- 업데이트: [[기존페이지]] (변경 내용 한 줄)
```

- 업데이트된 파일 목록
- 핵심 인사이트 1~2줄 (사용자 답변에서 특히 가치 있는 것)
